1. Jelaskan apa yang dimaksud dengan dokumen root ?
    Folder utama dimana semua file situs web disimpan. Juga lokasi dimana server web mencari file untuk ditampilkan ke pengguna.

2. Pada apache web server, apakah dokumen root harus terletak dalam direktori C:\xampp\htdocs ? Jelaskan jawaban anda
    Tidak harus, karena walaupun C:\xampp\htdocs merupakan lokasi default, kalian bisa memilih ke direktori lain asalkan kalian konfigurasi apache dengan benar.

3. Jelaskan apa yang dimaksud dengan perintah git : init, clone, remote, add, commit, push dan checkout
    - Git init = Untuk menginisialisasi repositori git baru di dalam direktori yang ada.
    - Git clone = Untuk membuat salinan lokal dari repositori yang ada.
    - Git remote = Untuk mengelola repositori jarak jauh (remote repositories).
    - Git add = Untuk menambahkan perubahan file tertentu atau semua file yang sudah diubah.
    - Git commit = Untuk menyimpan perubahan yang telah ditambahkan ke staging area ke dalam repositori disertai dengan pesan yang menjelaskan perubahan yang dilakukan.
    - Git push = Untuk mengirimkan commit yang telah dibuat di repositori lokal ke repositori jarak jauh.
    - Git checkout = Untuk berpindah antara cabang (branch) dalam repositori git.

4. Jelaskan perbedaan antara GitHub dan GitLab
    Github lebih dikenal sebagai platform untuk berbagi kode dan kolaborasi dengan fitur sosial yang kuat. Sedangkan gitlab menawarkan alat untuk seluruh siklus hidup pengembangan perangkat lunak, termasuk integrasi CI/CD yang lebih komprehensif.

5. Apa bedanya variable user dengan variable system ?
    Variabel user adalah variabel yang menyimpan informasi spesifik untuk sesi pengguna/aplikasi dan bisa diubah oleh pengguna. Sedangkan variabel system berisi informasi penting untuk operasi sistem secara keseluruhan dan biasanya tidak bisa diubah oleh pengguna. 

6. Apa persamaan variable system dengan variable constant ?
    Keduanya memiliki nilai yang tetap selama program berjalan, sering digunakan untuk konfigurasi dan dapat diakses di berbagai bagian program atau sistem.
